Code of Conduct

  • Respect every participant, mentor, volunteer, and venue staff—harassment of any kind is not tolerated.
  • Keep project workspaces tidy and share lab resources responsibly; return loaner hardware after each sprint.
  • Follow venue safety rules: badges visible at all times, no tailgating, and use designated exits during drills.
  • Report incidents immediately to the Kickstart team or emergency POCs; we will respond confidentially.
  • Ship original work. Plagiarism or presenting pre-built projects results in instant disqualification.

Need to flag something? Visit the operations desk beside the 2nd Floor Lab entrance.

Submission Guidelines

  1. Create or update a public Git repository by 16:00 with readable commits.
  2. Add a README covering problem statement, solution overview, setup steps, datasets, and team members.
  3. Upload your 5-slide deck and any demo videos to the shared Drive folder linked on Discord.
  4. Complete the submission form at the ops desk before 17:30 with repo, deck, and demo URLs.
  5. Be stage-ready by 17:45—bring laptop, cables, and confirm HDMI compatibility during tech check.

Missed a step? Ping the submission volunteers stationed near the seminar hall entrance.
