Tracks

  • Mobile apps for student life, services, or safety
  • AI features: assistants, routing, forecasting, anomaly detection

Hardware

(IoT solutions / Robotic solutions / Product-based solutions)

  • Sensor networks: occupancy, air quality, energy meters
  • Robotics for logistics, maintenance, or security demos
  • End-to-end prototypes: device + firmware + dashboard

Judging Criteria

Scorecards balance innovation, execution, and impact. Each judge reviews independently before alignment.

Impact & Relevance · 35%

Solves a real Smart Campus need with measurable outcomes.

Innovation · 25%

Creative use of data, hardware, and/or AI-driven experiences.

Technical Execution · 25%

Prototype depth, architecture decisions, code quality, deployability.

Storytelling & UX · 15%

Clarity, user journey, and demo polish within seven minutes.

Submission Guidelines

  1. Open-source your code.
  2. Share the repo URL and a README with setup, datasets, and features.
  3. Host your build if possible—live demos get extra credit.
  4. Submit deck, demo assets, and repo link before 17:30.
